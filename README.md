# ASTRASENSE ŌĆō AI-Powered Environmental Risk Intelligence
ASTRASENSE is a modern, interactive web application designed for **hyperlocal environmental intelligence**. It simulates satellite-derived indices (like NDVI, NDWI, LST, and EVI) and processes them into **AI-style hazard risk predictions** (such as floods, droughts, vegetation stress, and land degradation). 
The platform pairs statistical data visualizations with an interactive 3D globe and localized mapping interfaces, serving as a comprehensive dashboard for environmental monitoring.

## ­¤¦Ā Simulation Engine ([src/lib/sim/engine.ts](cci:7://file:///Users/pranavpatil/Documents/projects/astra%20copy/src/lib/sim/engine.ts:0:0-0:0))
Since this is a demonstration environment, ASTRASENSE uses a dynamic backend simulation engine rather than a live satellite ingestion pipeline.
*   **Index Generation:** The engine uses procedural probability algorithms mixed with geographic bounds to simulate realistic environmental data:
    *   **NDVI (Normalized Difference Vegetation Index):** Indicates the health and density of live, green vegetation.
    *   **NDWI (Normalized Difference Water Index):** Highlights open water features and moisture levels in vegetation.
    *   **LST (Land Surface Temperature):** Estimates surface thermal activity and urban heat islands.
    *   **EVI (Enhanced Vegetation Index):** Corrects for atmospheric and background canopy noise.
*   **Hazard AI Model:** The simulated indices are cross-referenced to trigger specific hazards. For example, high LST coupled with low NDWI increases the probability score for a **Drought** or **Wildfire**, while high NDWI increases the risk of a **Flood**.
*   **Temporal Trends:** The API generates up to 30 days of randomized historical timeline data to populate the interactive charts.

## ­¤¦® Component Architecture
The project follows a modular Next.js App Router structure:
*   **`src/app/`**: Contains the main application routes (`/dashboard`, `/map`, `/alerts`, `/about`).
*   **`src/app/api/`**: 
    *   `/geocode` ŌĆō A server-side OpenCage proxy to protect your API keys.
    *   `/simulate` ŌĆō The core API route that generates indices, hazards, alerts, trends, and hotspots based on requested coordinates.
*   **`src/components/`**:
    *   `/alerts` ŌĆō UI for the Alert Feed and severity-based Alert Cards.
    *   `/charts` ŌĆō Recharts-powered graphs for `HazardProbabilities` and `IndexTrends`.
    *   `/dashboard` ŌĆō Layout components including the `RiskPanel` and `IndicatorCard`.
    *   `/globe` & `/map` ŌĆō Complex spatial components (`AnalysisMap` and `GlobalGlobe`).
    *   `/location` ŌĆō Autocomplete fields (`LocationSearch`) and state management hooks (`useSelectedLocation`).
